## Components

### Backend (Node.js/Express)
- **Purpose:** Core orchestrator. Manages teams, event state, and submission processing.
- **Location:** `backend/`
- **Dependencies:** `express`, `mongoose`, `socket.io`, `jsonwebtoken`, `@google/genai`.

### CLIP Service (Python/Flask)
- **Purpose:** Standalone microservice for calculating visual similarity using the OpenAI CLIP model.
- **Location:** `clip_service/`
- **Dependencies:** `flask`, `torch`, `transformers`, `Pillow`.

### API Routes (`routes/api.js`)
- **Purpose:** Defines all REST endpoints for registration, login, submission, and admin controls.
- **Integration:** Uses JWT for authentication and Socket.io for real-time updates.

### AI Services (`services/`)
- **`ai.js`:** Wraps Gemini API calls for image generation and rubric-based evaluation.
- **`clip.js`:** Interface for communicating with the Python CLIP service.

## Data Flow

1. **Registration:** Team registers via `/api/register` → Stored in MongoDB → Receives `team_id` and JWT.
2. **Submission:** Team submits prompt via `/api/submit` → Server triggers:
    - Gemini Image Generation
    - CLIP Similarity Comparison (via Python service)
    - Gemini Image Evaluation (rubric)
3. **Scoring:** Scores are blended (60% CLIP, 40% Gemini) → Updated in MongoDB → Broadcasted via Socket.io.
4. **Leaderboard:** Clients receive `leaderboard_update` event → Fetch new data and refresh UI.
